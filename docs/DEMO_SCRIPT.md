## 1. 데이터 아키텍처 (40초)

**화면**: 대시보드에서 [아키텍처 뷰] 탭 클릭

> **전체 데이터 흐름**입니다. 상단에 EMR·처방·검사·영상·원무 등 **정형 데이터**와 PACS·병리 등 **비정형 데이터** 원천 시스템이 있습니다.

**화면**: IDP 파이프라인 영역 가리키기

> **CDC 실시간 동기화**로 데이터를 수집하고, **정형·공통·비정형 3단계 처리**를 거칩니다. 처리된 데이터는 **Data Lakehouse** — 원본 보존, 정제·검증, 국제표준 변환 — 3개 Zone에 단계별로 적재됩니다.
>
> 아래 포털에서 **BI 대시보드**, **AI 분석환경**, **의무기록 분석**, **자연어 데이터 조회** 4가지 방식으로 연구자에게 제공됩니다. 하단에 **비식별화**, **접근 제어** 등 거버넌스 체계도 갖추고 있습니다.

---

## 2. 데이터 제공 유연성 (50초)

**화면**: `/cdw` 이동 → 미리 실행된 예시 질의 결과 화면

> 교수님이 데이터를 원하실 때 **세 가지 방식**을 제공합니다.
>
> 첫째, **자연어 질의**입니다. 한국어로 질문하면 AI가 SQL을 자동 생성하고 실행합니다. IT 부서를 거치지 않고 직접 조회할 수 있습니다.

**화면**: 결과의 사고 과정 펼치기 → 생성된 SQL · 실행 결과 확인

> 결과에 AI의 사고 과정, 생성된 SQL, 실행 결과가 함께 제공됩니다.
>
> 둘째, **BI 대시보드**로 반복 조회를 시각화할 수 있고, 셋째, **JupyterLab** 분석환경으로 Python/R 코드 기반 심층 분석도 가능합니다. 기술 수준에 따라 유연하게 선택할 수 있습니다.

---

## 3. AI (40초)

**화면**: `/medical-knowledge` → 미리 실행된 "수근관 증후군 진단" 검색 결과

> **의학 지식 검색**입니다. "수근관 증후군 진단"을 검색하면 AI가 영문 의학 용어로 자동 확장하여 **50만 건** 의학 문헌에서 관련 근거를 찾아줍니다.

**화면**: `/ontology` 이동 → 3D 지식 그래프 화면

> **의료 온톨로지**입니다. 테이블 간 관계와 표준 용어 체계가 지식 그래프로 시각화되어, 데이터 구조를 직관적으로 파악할 수 있습니다.

**화면**: `/ner` 이동 → 미리 실행된 당뇨 진료기록 분석 결과

> **의무기록 자동 분석**입니다. 임상 텍스트에서 진단·약물·검사를 자동 추출하고 국제 표준 코드에 매핑합니다.

---

## 4. Mart 구성 (40초)

**화면**: `/datamart` 이동 → [임상 데이터 현황] 탭 (기본)

> 지금까지 보신 기능이 동작하는 **실제 데이터마트**입니다. 환자 인구통계 성별 분포, 방문 유형별 입원·외래·응급 분포, 도메인별 데이터 품질 점수를 확인할 수 있습니다.

**화면**: [의료 영상] 탭 클릭

> **의료 영상**입니다. 흉부·유방촬영 등 부위별 X선 영상 **28만 장**과 어노테이션 데이터가 적재되어 있습니다.

**화면**: [테이블 관리] 탭 → person 테이블 선택

> 테이블을 선택하면 **스키마 정보**, **샘플 데이터**, **Python/R 사용 예제 코드**를 바로 조회할 수 있습니다.
