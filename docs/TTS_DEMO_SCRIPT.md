## 1. 홈 대시보드

[전환: /dashboard — 홈]

먼저 홈 화면입니다.
플랫폼에 접속하면 가장 먼저 보이는 플랫폼 현황 대시보드입니다.

상단 헤더 바에는 통합 검색창이 항상 표시되어 있어,
테이블명이나 컬럼명을 입력하면 데이터 카탈로그로 바로 이동합니다.

[잠시]

대시보드 상단에는 세 가지 뷰 모드 전환 버튼이 있습니다.
레이크하우스, 운영 뷰, 아키텍처 뷰입니다.
옆에는 레이아웃 편집과 리포트 내보내기 버튼도 제공됩니다.

바로 아래에는 시스템 상태 배너가 표시되어,
현재 시스템 온라인 여부, CPU 사용률, 메모리 사용량을 실시간으로 확인할 수 있습니다.

[잠시]

기본 뷰인 레이크하우스 뷰를 살펴보겠습니다.
상단에 네 개의 KPI 카드가 있습니다.
총 데이터 자산 수와 전체 레코드 수, 등록 환자 수, CDW 임상 테이블 수, EDW 운영 테이블 수를 한눈에 보여줍니다.

[잠시]

중간 영역에는 세 개의 카드가 나란히 표시됩니다.
왼쪽의 도메인별 데이터 분포 파이차트에서는 Clinical, Lab, Demographics 등 도메인별 데이터 비중을 확인할 수 있고,
가운데 진료유형별 분포 바차트에서는 입원, 외래, 응급 등 방문 유형별 건수를 보여줍니다.
오른쪽 추천 데이터셋에서는 활용도가 높은 테이블을 클릭하여 카탈로그로 바로 이동합니다.

하단에는 데이터 맵 트리맵과 전체 데이터 자산 목록 테이블이 있어,
각 테이블의 도메인, 데이터 규모, 비중을 상세히 탐색할 수 있습니다.

### AI 도우미

[전환: 헤더 바 우측 AI 도우미 버튼 클릭]

이제 상단 헤더 바 오른쪽에 있는 AI 도우미 버튼을 클릭해 보겠습니다.
화면 오른쪽에서 민트색 채팅 패널이 열립니다.

[잠시]

AI 도우미는 현재 보고 있는 페이지의 맥락을 자동으로 인식합니다.
자연어로 질문하면 LLM이 답변을 생성하고, SQL 쿼리 결과를 테이블로 바로 보여줍니다.

예를 들어, "폐렴 소견 흉부 X-ray 보여줘"라고 입력하면,
AI가 해당 의료 이미지를 대화창 안에 직접 표시해 줍니다.

[잠시]

답변 아래에는 추천 액션 버튼이 나타납니다.
이 버튼을 클릭하면 데이터 카탈로그나 관련 페이지로 바로 이동할 수 있어,
대화 흐름에서 곧바로 데이터 탐색으로 연결됩니다.

대화 기록 버튼을 누르면 이전 대화를 타임라인 형태로 확인하고,
원하는 시점으로 복원하면 당시 보고 있던 화면까지 함께 되돌아갑니다.

AI 도우미는 어떤 페이지에서든 헤더 바를 통해 항상 접근할 수 있습니다.

### 2-1. 운영 뷰

[전환: 운영 뷰 버튼 클릭]

운영 뷰에서는 실시간 시스템 상태를 모니터링합니다.

[전환: 아키텍처 뷰 버튼 클릭]

상단에는 EMR, OCS, LIS, PACS, ERP 등 원천 시스템이 있고,
CDC와 FHIR을 통해 데이터가 수집됩니다.
중간의 데이터 레이크 영역을 거쳐,
하단의 BI 대시보드, AI 분석 환경, NLP 구조화, CDW 연구 지원 서비스로 제공됩니다.
아래에는 하드웨어 배치 구성안과 포트 매핑 정보도 함께 표시됩니다.

## 5. 데이터 활용

### 5-1. 데이터마트

[전환: /datamart]

데이터마트 화면입니다.
OMOP CDM 버전 6 기반의 임상 데이터를 탐색하고 관리합니다.
현재 약 9,200만 건의 레코드와 76,000명의 환자 데이터가 적재되어 있습니다.

#### CDM 변환 요약

[전환: CDM 변환 요약 탭]

CDM 변환 요약 탭입니다.
총 환자 수, 총 레코드, CDM 테이블 수, 평균 연령이 상단 카드에 표시됩니다.

환자 인구 통계에서는 성별 분포를 파이차트로 확인하고,
방문 유형 분포에서는 입원, 외래, 응급 비율을 바차트로 보여줍니다.
도메인별 데이터 품질 점수도 프로그레스바로 표시됩니다.

연도별 활동 추이 차트와 주요 진단 상위 15개 목록,
그리고 소스에서 OMOP CDM으로의 매핑 예시 테이블이 있어
표준 변환 과정을 상세히 확인할 수 있습니다.

#### 테이블 관리

[전환: 테이블 관리 탭]

테이블 관리 탭입니다.
왼쪽에서 OMOP CDM 테이블을 검색하고 선택하면,
오른쪽에 해당 테이블의 상세 정보가 표시됩니다.

스키마 정보 탭에서는 컬럼별 타입과 Nullable 여부를 확인하고,
샘플 데이터 탭에서는 실제 데이터를 미리 볼 수 있습니다.
사용 예제 코드 탭에서는 파이썬과 R 언어로 바로 사용 가능한 코드를 제공합니다.
설명 수정과 CSV 내보내기 기능도 지원합니다.

### 6-2. 비정형 데이터 구조화 — Medical NER

[전환: /ner]

비정형 데이터 구조화 화면입니다.
의료 텍스트에서 진단명, 약물명, 검사명 등의 개체명을 자동 인식하는 NER 시스템입니다.

[잠시]

왼쪽에 의무 기록 텍스트를 입력하고 분석 실행 버튼을 누르면,
오른쪽에 인식 결과가 표시됩니다.

원문에서 추출된 개체명이 색상별로 하이라이트 되고,
진단은 빨간색, 약물은 파란색, 검사는 초록색, 시술은 보라색으로 구분됩니다.

아래에는 추출된 엔티티 목록이 테이블로 정리되며,
각 엔티티에 대한 OMOP Concept 매핑과 SNOMED CT, RxNorm, LOINC 표준 코드가 자동 부여됩니다.
코드 매퍼 섹션에서는 코드 시스템별로 매핑 결과를 분류하여 보여주고,
PII 탐지 경고도 함께 제공합니다.

한국어와 영어 의료 텍스트를 모두 지원하며,
GPU 서버의 BioClinicalBERT 모델을 기반으로 동작합니다.
