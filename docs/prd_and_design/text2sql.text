맞다. 정확히 이해했다.

## 그림 해석

이 이미지는 **엔코아 BizMeta + ITMeta** 아키텍처인데, 핵심은:

```
┌─────────────────────────────────────────────────────────┐
│                    LLM 질의                             │
│              "작년 위암 환자 수"                         │
└─────────────────┬───────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────┐
│              RAG (Vector Search)                        │
│                                                         │
│  ┌───────────────────────────────────────────────────┐ │
│  │           MDR-Biz (BizMeta)                       │ │
│  │  • 업무용어 → 보고서/데이터 매트릭스              │ │
│  │  • Biz용어 = 기준 데이터 명                       │ │
│  │  • "위암" → ICD C16 → 관련 테이블                │ │
│  └───────────────────┬───────────────────────────────┘ │
│                      │ Mapping                          │
│  ┌───────────────────▼───────────────────────────────┐ │
│  │           MDR-Tech (ITMeta)                       │ │
│  │  • IT용어 → Table → DBMS/System                  │ │
│  │  • 표준용어 → 개발용어                            │ │
│  │  • cancer_patients.cancer_type = 'stomach'       │ │
│  └───────────────────────────────────────────────────┘ │
└─────────────────────┬───────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────┐
│                    LLM SQL 생성                         │
│  Context = BizMeta + ITMeta + Schema                   │
│                                                         │
│  SELECT COUNT(*) FROM cancer_patients                   │
│  WHERE cancer_type = 'stomach' AND YEAR(...) = 2025    │
└─────────────────────────────────────────────────────────┘
```

## DS 방침 요약

```
LLM → RAG(BizMeta + ITMeta) → LLM(SQL생성)
```

**핵심:**
- **BizMeta** = 비즈니스 용어 사전 (의료 용어 → 표준 코드)
- **ITMeta** = 물리 스키마 매핑 (테이블, 컬럼, 관계)
- 이 메타데이터를 **RAG 컨텍스트**로 LLM에 제공

이게 DS가 말하는 "BizMeta-ITMeta 연동하면 정확도 높아진다"의 실체다.
